# 火影忍者手游项目 — 资产管理 Skill

## 资源分类定义（最重要！）

| 目录 | 内容 | 用途 |
|------|------|------|
| **立绘序列** | 角色原画大图，PNG序列帧 | **最常用素材**，大图展示 |
| **3D动画序列** | 游戏内3D模型渲染的动作动画 | 模拟游戏视频制作，包含 attack/hit/idle/run/skill01/skill02/skill03 等动作目录 |
| **技能特效序列** | 角色技能特效动画 | 独立存放于 `技能特效序列/[角色名]/` |

> **立绘动画序列** = 立绘序列（命名不统一，统一按立绘序列处理）

```

```

## 角色命名规则

### 格式：`[角色名][状态/形态]`

| 组件 | 说明 | 示例 |
|------|------|------|
| 角色名 | 角色名称 | 奈良鹿丸、佐助、鸣人 |
| 状态/形态 | 觉醒、少年、限定等 | 佐助**觉醒**立绘、**少年**佐助、**限定**鸣人 |

### 关键区分维度

**① 品级不同 = 不同角色（必须区分）**
- 鸣人 ≠ 限定鸣人（不同品级，是两个角色）
- 命名中含"限定"= 限定品级角色

**② 状态不同 = 同一角色的不同阶段（独立存放）**
- 佐助立绘（默认形态）
- 佐助觉醒立绘（觉醒状态）
- 少年佐助（少年形态）

**③ 别名处理原则**
- "立绘动画序列" = "立绘序列"（同类资源，统一按立绘序列处理）
- 遇到别名时 → 替换为标准名称（见下方角色名称标准）
- 遇到不明确名称 → 保留原名 + 在汇报中标注疑问

### 角色名称标准

**优先使用全称，不得擅自缩写。**

| 缩写/别名 | 标准名 |
|---------|--------|
| 鹿丸 | 奈良鹿丸 |

遇到别名时，统一替换为标准名。如有疑问，在汇报中标注。

### 忽略的命名元素
- `K7` 等研发内部代号 → **不需要管**，不作为分类依据

## 收到新资源包后的处理流程

### Step 1：下载
```bash
bdpan download "<分享链接>" <本地路径> -p <提取码>
```
或直接给网盘链接+提取码，让 Agent 执行下载。

### Step 2：分析 ZIP 内容
- 列出 ZIP 内文件结构
- 识别每个文件/文件夹对应的角色、状态、资源类型
- 判断是否有重复（对比现有目录）

### Step 3：归类决策

**判断逻辑：**
```
资源类型 = 立绘 / 3D动画 / 技能特效？
├── 立绘类 → 立绘序列/[角色名][状态]/
├── 3D动画类 → 3D动画序列/[角色名][状态]/
├── 技能特效类 → 技能特效序列/[角色名]/
└── 通用特效 → 技能特效序列/（根目录）

状态判断：
├── 文件名/文件夹含"觉醒" → 觉醒状态
├── 含"少年" → 少年形态
├── 含"限定" → 限定品级角色
└── 无标注 → 默认形态

是否重复？
├── 同名文件已存在 → 跳过（不覆盖）+ 在汇报中标注
└── 新角色/新状态 → 正常放入
```

### Step 4：执行移动/解压
```bash
# 解压到临时目录
ditto -x -k --sequesterRsrc <zip路径> /tmp/huoying-extract/

# 移动到目标位置（注意中文路径处理）
mv /tmp/huoying-extract/<内容> "/Volumes/项目/项目_火影/美术资源/<目标目录>/"

# 清理
rm -rf /tmp/huoying-extract/
```

### Step 5：汇报通知（直接发给用户）

**固定格式：**
```
共享盘已更新 [角色名] [资源类型描述] 到对应文件夹

路径：\\rygx.com\项目\项目_火影\美术资源\<实际目录>
内容：[立绘动画 X 张] / [3D动画 X 套] / [技能特效 X 套]
状态：[默认形态] / [觉醒状态] / [少年形态]
备注：[如有重复文件已跳过] / [如有命名疑问已标注]
```

**示例：**
```
共享盘已更新 奈良鹿丸 立绘动画序列 到对应文件夹

路径：\\rygx.com\项目\项目_火影\美术资源\立绘序列\奈良鹿丸
内容：立绘动画60张（idle序列）
状态：默认形态
备注：原包中"鹿丸"已统一为"奈良鹿丸"
```

## 核心约束

1. **不覆盖已有文件** — 遇到重名文件跳过，不自动覆盖
2. **不删除任何文件** — 仅新增/归类，不做清理
3. **不确定时先问** — 遇到无法判断的角色/状态，先向用户确认再执行
4. **中文路径用 ditto** — 解压中文名文件不用 unzip，用 `ditto -x -k`
5. **路径使用 SMB 格式展示** — 汇报时将 `/Volumes/项目/项目_火影/` 转换为 `\\rygx.com\项目\项目_火影\`

## 遇到以下情况必须先确认

- 同一角色不同版本（是否合并/并存？）
- 资源类型无法判断（先描述内容，用户确认分类）
- 目标目录已有同名文件夹（是否合并？是否覆盖？）
